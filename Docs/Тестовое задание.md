Для выполнения задания используйте Unity 2022.3, последнюю из LTS.

Описание:

Создайте небольшой прототип игры от первого или от третьего лица в Unity,

в котором игрок управляет персонажем с возможностью стрелять в противников.

Противники должны иметь базовый искусственный интеллект и показатель здоровья (hp).

  

У игрока должно быть:

1. Управление персонажем с помощью клавиатуры и мыши, с выстрелом по кнопке.

Важно: управление должно быть имплементацией под компьютер и предусматривать возможность быстрого изменения под мобильные устройства (с джойстиком и кнопкой выстрела на экране).

2. Отображение здоровья игрока с помощью UI. Например, полоска HP в верхнем левом углу экрана).

3. Игрок имеет 3 улучшаемые способности (характеристики): скорость, здоровье, урон.

4. За каждого убитого врага игрок получает 1 очко улучшения характеристик, за которые он сможет улучшить показатель характеристики.

5. Изменение характеристик должно сказываться на игроке (персонаж быстрее бегает, у него больше здоровья на полоске HP, он наносит больший урон).

  

Интерфейс:

Создайте интерфейс окна, в котором игрок может улучшать способности своего персонажа. Переход в это окно должен быть по кнопке на экране игрока.

В окне должно быть:

1. Отображение доступного количества очков улучшений;

2. Список характеристик;

3. Для каждой характеристики должны быть:

 - Индикатор текущего уровня (полоска или число).

 - Кнопка улучшения ("+").

4. Кнопка Применить. По нажатию способности применяются и окно закрывается.

5. Кнопка выхода из окна без применения улучшений ("x").

  

Логика улучшений:

При нажатии на кнопку "+" у выбранной характеристики визуально увеличивается уровень, а очки уменьшаются. Важно: только визуально увеличиваются, применяются к игроку очки только после кнопки Применить.

Когда очки заканчиваются, кнопки "+" становятся неактивными.

  

Общие рекомендации:

- 3D игровой мир.
    
- Простой набор ассетов (можно использовать примитивы Unity: кубы, сферы, либо сторонние ассеты).
    
- Для управления игроком можно использовать стандартные ассеты Unity, но в игре должна быть заложена возможность перехода от управления кнопками к мобильному тачскрину.
    
- Логика противников может быть совсем примитивной - просто ходить по случайной траектории или даже стоять.
    
- Стартовое здоровье противников случайно, можно задать в пределах 1-10 не улучшенных выстрелов игрока.
    
- Выводить полоску hp врагам не обязательно, но умирать при окончании здоровья они должны.
    
- Окружение сцены не обязательно, можно просто на плейне двигаться.
    
- По возможности добавьте базовые эффекты (например, вспышка выстрела, звук, анимация исчезновения противника).
    
- Предусмотрите возможность масштабирования (как минимум, добавление новых характеристик в следующих обновлениях).
    
- Реализуйте адаптивный интерфейс, чтобы окно корректно отображалось на разных экранах.
    
- Качественное изменение характеристики за каждое очко улучшений, а также максимальные ограничения характеристик, должны конфигурироваться в таблице (к примеру каждое очко улучшений вложенное в бег добавляет 5% к скорости игрока, но не более 200%).
    
- Желательно, чтобы прогресс улучшения игрока сохранялся между сессиями.
    

  

Приоритеты оценки тестового задания:

- Код должен предусматривать масштабирование игры для будущих обновлений; 
    

- Возможность локализации (можно без самой системы);
    
- Получение конфигураций из разных источников (в игре может быть 1 источник из ScriptableObject);
    
- Использование DI контейнеров (Zenject и т.п.), UniRx является плюсом;
    
- Необработанные крайние состояния, краши и ошибки при исполнении игры - большой минус.
    

К визуальной части главное требование чтобы все правильно выглядело на любом разрешении экрана.
